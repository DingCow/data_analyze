当前 Web 页面抽象出的框架方向

这一步先定结构，不定视觉细节。下面的布局都基于你现在的 Streamlit 页面职责拆分出来。

现状结构抽象
顶部：标题 / 数据源状态 / 清空会话
主输入区：输入问题 + 提交按钮
分析过程区：消息线程 / 过程摘要
结果工作区：摘要卡 / 图表 / 数据表 / 推荐追问
A. 保守演进型
Header
Input Tray
Process Stream
Result Workspace

A. 保持当前阅读顺序

仍然是从上往下读：先输入，再看过程，再看结果。最稳，改动最小，也最贴近你现在的页面。

B. 分析工作台型
Header
左侧：过程 / 历史
右侧：结果工作区
底部：统一输入托盘

B. 更像桌面分析台

把“过程”和“结果”并排，输入区固定成底部托盘。更像专业分析工具，我更推荐这个方向。

C. 仪表盘首页型
Hero Header
Quick Card
Quick Card
Quick Card
Input
Results

C. 首页感更强

更像仪表盘产品首页，视觉会更活，但会偏离你现在“分析工作流”导向的页面节奏。

我当前推荐

  • 优先选 B：最像“分析工作台”，也最适合后续继续做 Pencil 低保真框架。
  • 如果你希望先稳住，再小步改，就选 A。

这一步先不处理

  • 先不讨论颜色、玻璃质感、阴影和字体。
  • 先不处理图表细节和组件皮肤。
