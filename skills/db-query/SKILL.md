# DB Query

快速查询项目 SQLite 数据库中的单表信息，并用结构化方式展示结果。

---

## 使用边界

- 默认数据库固定为 `/Users/owenlau/SqliteDB.db`
- 只做只读查询，不做写操作
- 优先用于：
  - 看某张表有哪些字段
  - 看最近几条样例数据
  - 看表总行数
  - 帮用户快速确认表内容是否符合预期

---

## 执行步骤

### Step 1：确认查询目标

从用户请求中提取：

- 表名
- 需要查看的条数，默认 `20`
- 是否需要额外查看字段结构或总行数

如果用户没有说清楚表名，先追问表名，不要猜。

### Step 2：确认表是否存在

先检查目标表是否在数据库中存在。

推荐命令：

```bash
sqlite3 /Users/owenlau/SqliteDB.db ".tables"
```

如果表不存在，直接告诉用户，不继续查样例数据。

### Step 3：读取表结构

查询字段信息，确认列名和类型。

推荐命令：

```bash
sqlite3 /Users/owenlau/SqliteDB.db "PRAGMA table_info(<TABLE>);"
```

### Step 4：读取样例数据

优先查询最近若干条记录。
如果表没有明显时间字段，就按 `rowid DESC` 取最近数据。

推荐命令：

```bash
sqlite3 /Users/owenlau/SqliteDB.db "SELECT * FROM <TABLE> ORDER BY rowid DESC LIMIT <LIMIT>;"
```

### Step 5：读取总行数

如果用户关心数据规模，或样例数据需要附带上下文，则补查总行数。

推荐命令：

```bash
sqlite3 /Users/owenlau/SqliteDB.db "SELECT COUNT(*) FROM <TABLE>;"
```

### Step 6：结构化输出

输出时尽量使用：

- 表格展示字段结构
- 表格展示样例数据
- 一行总结说明总行数、抽样条数、是否发现空结果

---

## 输出要求

- 明确说明查询的是哪张表
- 明确说明使用的是固定数据库路径
- 字段结构和样例数据分开展示
- 如果结果为空，直接说明“表存在，但当前查询结果为空”
- 如果字段很多，样例数据可以只展示关键列，并说明已做截断
